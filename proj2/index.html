Task 1

De Casteljau’s algorithm converts a series of points into a smooth curve. This is done through a recursive algorithm where, at every step, every pair of neighboring points is linearly interpolated through some t∈[0,1]t \in [0, 1]t∈[0,1]﻿ to create a new point. These new points from each neighboring pair are then used in the next iteration of the algorithm until there is a single point left. To complete the entire curve, this algorithm is performed over a sweep of ttt﻿ values.

We implemented this as a function that took in a series of nnn﻿ Vector2D points, and returned the next step in the algorithm: a series of n−1n-1n−1﻿ points.

Task 2

To use de Casteljau’s algorithm for a 2D grid of points, one can reuse the recursive algorithm for the 1D case. This is done by calculating the final point for every row in the grid. This results in nnn﻿ points that can then be run again through de Casteljau’s algorithm. In order to sweep through the whole space, we need two scalar interpolation parameters: uuu﻿ for the row direction and vvv﻿ for the column direction.

Task 6

We implemented loop subdivision by first computing the updated vertex positions for existing vertices in the input mesh. We used the approach described in the spec by where we use a weighted combination of the original vertex position and the sum of each of it’s neighbor’s original positions based its degree. We then updated the vertex position associated with each edge (splitting the edge) using the scheme described in the spec, setting the edge’s newPosition. To actually perform the subdivision, we iterated through each of the edges in the input mesh and split it, updating the returned vertex’s newPosition with the newPosition from the edge that it split. We then looped through all edges in the final mesh, flipping any edge with our flipEdge implementation that was added.

To account for newly added vertices and edges, we modified our Task 5 implementation to specify that edges (A, M) and (M, D) were new edges and that vertex M was new as well.

There was not much debugging necessary for this task. Each partner wrote a separate implementation and while each implementation had minor issues (a couple of edges flipped the wrong way causing the surface to look uneven), we found that after merging each of our implementations that these issues resolved themselves.

Results:

Failures

With models with edges, there is some jagginess. This is because our subdivision method immediately returns if the edge is next to a boundary. Therefore, the resulting subdivided model has undivided faces near its edges.

To fix this, we can pre-split the boundary edges, and then continue with subdivision. This helps partially to smooth out these areas. This can be further smoothed by implementing boundary edge subdivision.

With objects with sharp corners, subdividing drastically changes the volume and shape of the mesh. This is because those edges that jut out significantly get averaged by the points around it. This can be also be alleviated by pre-splitting some edges, as this adds additional points around sharp corners that will lessen the averaging effect.
